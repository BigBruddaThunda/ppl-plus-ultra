**CSS Arbitration Spec** (`.planning/phases/04-design-tokens/04-CSS-ARBITRATION.md`):

Write the exhaustive CSS property-to-dial-owner mapping document. This is the single authority for "which dial owns which CSS property." No property has two owners.

Structure: A markdown table with columns: CSS Custom Property Pattern | Owner Dial | Value Source | Count | Notes

**Ownership model (locked in CONTEXT.md — 3+1 model):**

**Order (structural):** fontWeight, fontWeightDisplay, lineHeight, spacingMultiplier, letterSpacing, density, animationDuration, opacity (default states), gridGap/blockGap. Principle: rhythm, density, weight.

**Color (tonal):** primary, secondary, background, surface, text, accent, border, borderRadius, shadowDepth, saturation. Principle: mood, intensity, palette.

**Axis (directional):** gradientDirection, layoutFlow, fontFamily, typographyBias. Principle: direction, character, visual flow.

**Type (content/icon):** Reserved slot for body-region illustrations, muscle group icons, movement pattern diagrams. NOT layout or color — purely visual content. Phase 4 declares the slot; actual illustrations are deferred per CONTEXT.md.

Include the complete mapping table from the RESEARCH.md Pattern 3, expanded with edge-case assignments from Pattern 6. Add a "Principle" section explaining the single rule: intensity/mood -> Color, rhythm/density/weight -> Order, direction/character -> Axis.

Include the W enum position -> tonal name bridge table (W.TEACHING=19 -> 'order', W.BODYWEIGHT=20 -> 'growth', etc.) so Phase 5 has the authoritative mapping.
