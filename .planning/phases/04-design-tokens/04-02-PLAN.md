**Step 1: Create the color derivation script** (`canvas/scripts/derive-colors.mjs`).

This ESM script uses culori to derive 8 Color palettes from base hue parameters. For each Color:
- Define base hue (H), base chroma (C), and primary lightness (L) based on intaglio aesthetic
- Derive 7+ palette properties using the formulas from RESEARCH.md Pattern 2
- Apply `toGamut('p3')` to every derived value for gamut safety
- Format each value as `oklch(L C H)` string using culori's `formatCss()`

Base hue table (starting values from research, tuned for intaglio/banknote aesthetic):
```
order:        h=0,   c=0.00, L=0.20  (⚫ near-achromatic black ink)
growth:       h=145, c=0.12, L=0.42  (🟢 forest green, not neon)
planning:     h=230, c=0.14, L=0.40  (🔵 deep Prussian blue)
magnificence: h=285, c=0.18, L=0.38  (🟣 deep violet, GOLD color)
passion:      h=25,  c=0.22, L=0.42  (🔴 vermillion ink, GOLD color)
connection:   h=45,  c=0.18, L=0.55  (🟠 warm amber)
play:         h=85,  c=0.16, L=0.72  (🟡 brass yellow)
eudaimonia:   h=60,  c=0.02, L=0.92  (⚪ cream white, near-achromatic)
```

Palette derivation formulas per color:
- primary:    oklch(L, C, H)
- secondary:  oklch(L+0.10, C*0.80, H)
- background: oklch(0.97, C*0.10, H)
- surface:    oklch(1.00, 0, 0)
- text:       oklch(L-0.14, C*0.65, H)
- accent:     oklch(L-0.06, C*0.90, H)
- border:     oklch(0.82, C*0.30, H)

The script should output JSON to stdout or write directly to canvas/src/tokens/design-tokens.json.

**Step 2: Author the complete design-tokens.json.**

The JSON structure has three top-level sections: `colors`, `orders`, `axes`.

`colors` section: 8 entries keyed by tonal name. Each entry has metadata (emoji, scl_name, tonal_name, base_hue, base_chroma, tier, gold) plus the derived palette properties. The derive-colors.mjs script produces the palette values; metadata is hardcoded.

`orders` section: 7 entries keyed by ORDERS slug. Each entry has:
- emoji: string
- fontWeight: number (300-900 range, distinct per Order)
- fontWeightDisplay: number (one step heavier than fontWeight for headings)
- lineHeight: number (1.35-1.85 range)
- spacingMultiplier: number (0.85-1.50 range)
- letterSpacing: string (e.g., "0.005em" to "0.04em")
- density: string ("compact" | "moderate" | "spacious" | "airy")

Order typography values (distinct variation per Order, per research recommendation):
```
foundation:   fw=400, fwd=500, lh=1.75, sm=1.25, ls="0.02em",  density="spacious"
strength:     fw=800, fwd=900, lh=1.40, sm=0.85, ls="0.005em", density="compact"
hypertrophy:  fw=600, fwd=700, lh=1.55, sm=1.00, ls="0.01em",  density="moderate"
performance:  fw=900, fwd=900, lh=1.35, sm=0.75, ls="0",       density="compact"
full-body:    fw=500, fwd=600, lh=1.60, sm=1.10, ls="0.015em", density="moderate"
balance:      fw=400, fwd=500, lh=1.65, sm=1.15, ls="0.02em",  density="moderate"
restoration:  fw=300, fwd=400, lh=1.85, sm=1.50, ls="0.04em",  density="airy"
```

`axes` section: 6 entries keyed by AXES slug. Each entry has:
- emoji: string
- gradientDirection: string (CSS gradient direction)
- layoutFlow: string (CSS flex-direction value)
- typographyBias: string (serif/sans-serif/monospace character hint)

Axis gradient values (from research Pattern 4):
```
basics:     gradient="to bottom",      flow="column",         typBias="classical"
functional: gradient="135deg",         flow="row",            typBias="condensed"
aesthetic:  gradient="to bottom right", flow="column-reverse", typBias="editorial"
challenge:  gradient="to top",         flow="column",         typBias="industrial"
time:       gradient="to right",       flow="row",            typBias="monospace"
partner:    gradient="radial",         flow="row",            typBias="humanist"
```

**Step 1: Create style-dictionary build script** (`canvas/src/tokens/build/build-tokens.mjs`).

ESM script using style-dictionary v4 API:

1. Register custom name transform `name/ppl/css-var` that produces double-hyphen naming:
   - Token path `['colors', 'passion', 'primary']` -> CSS var name `ppl-color-passion--primary`
   - The transform strips trailing 's' from the category ('colors' -> 'color', 'orders' -> 'order', 'axes' -> 'axis')
   - The double-hyphen separates the group name from the property name

2. Configure two platforms:
   - `css`: transformGroup 'css' with custom name transform override, outputs `canvas/src/tokens/design-tokens.css` as `:root` CSS variables
   - `typescript`: outputs `canvas/src/tokens/tokens.ts` as a typed const export

3. For the TypeScript output: style-dictionary's built-in `javascript/esm` format may not produce the exact shape needed. If it does not produce a clean typed export, use a custom format that:
   - Exports `const tokens = { colors: {...}, orders: {...}, axes: {...} } as const`
   - Exports type aliases: `TonalColorName`, `OrderName`, `AxisName`
   - Exports the `COLOR_W_TO_TONAL` bridge table mapping W enum positions (19-26) to tonal name strings

4. Await `sd.hasInitialized` before building (critical v4 pitfall).

5. Add a `build:tokens` script to canvas/package.json: `"build:tokens": "node src/tokens/build/build-tokens.mjs"`
