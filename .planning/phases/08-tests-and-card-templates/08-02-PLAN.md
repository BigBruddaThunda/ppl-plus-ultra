    Define CardData and BlockData interfaces as specified in RESEARCH.md:
    ```typescript
    export interface CardData {
      zip: string;          // "⛽🏛🪡🔵"
      numericZip: string;   // "2123"
      operator: string;     // "🤌 facio"
      title: string;        // workout title from filename
      subtitle: string;     // derived from Order/Axis/Type/Color names + time estimate
      intention: string;    // 🎯 INTENTION text
      blocks: BlockData[];
      cssVars: Record<string, string>; // from weightsToCSSVars()

    }
    export interface BlockData {
      number: number;
      emoji: string;
      name: string;
      content: string;      // raw markdown block body
      contentHtml: string;  // parsed HTML for rendering
    }
    ```

    **Step 2 — Create `canvas/components/WorkoutCard.tsx`:**
    A PURE FUNCTION (not a React component) that takes CardData and returns an HTML string. The .tsx extension honors the locked decision for "TSX production component" while the pure-function approach avoids installing React in canvas/.

    ```typescript
    export function renderWorkoutCard(data: CardData): string { ... }
    ```

    The function produces the zoned layout per user decision:
    - **Header zone:** Title flanked by Type emoji (bold, large), subtitle, CODE line (zip), 🎯 intention (quoted)
    - **Block zone:** Each block in a uniform `.ppl-block` container with emoji header and parsed HTML content. Heavy border separators (═══ visual) between blocks.
    - **Footer zone:** 🚂 Junction block + 🧮 SAVE block

    CSS vars injected as inline `style` attribute on the `.ppl-card` container element (NOT on :root — per Pitfall 3 in RESEARCH.md). Format: `style="--ppl-theme-primary: oklch(...); --ppl-weight-saturation: 0.50; ..."`

    Also create a helper function to parse a .md card file into CardData:
    ```typescript
    export function parseCardFile(filePath: string): CardData { ... }
    ```
    Uses gray-matter for frontmatter extraction. Splits block body on `/^═══$/m` (U+2550 Box Drawings Double Horizontal, not standard ---). Uses marked for markdown-to-HTML conversion of block content. Derives numericZip from the emoji zip using the zip converter. Derives cssVars by calling resolveZip() + weightsToCSSVars(). Derives colorSlug from the Color position.

    CRITICAL: No hardcoded color values anywhere. All colors come from CSS custom properties (CARD-02).

    **Step 3 — Create `canvas/components/WorkoutCard.css`:**
    Intaglio art direction CSS (CARD-03). This is the banknote engraving aesthetic.

    Base card styles:
    - `.ppl-card` — Vignette edge darkening via `radial-gradient(ellipse at center, transparent 60%, rgba(0,0,0,0.08) 100%)`. Line-work border: `border: 2px solid var(--ppl-theme-border)`. Inner border: `outline: 1px solid var(--ppl-theme-border); outline-offset: -4px` for double-engraved look.
    - `.ppl-card__header` — Guilloche SVG background pattern using `--ppl-theme-border` color for stroke. Background via inline SVG data URI (concentric circles, opacity 0.10-0.15).
    - `.ppl-card__header-title` — Bold emoji flanking the title, font-size driven by `var(--ppl-weight-font-weight)`.
    - `.ppl-card__block-zone` — Scrollable container.
    - `.ppl-block` — Uniform container for all 22 block types. Hatching pattern as background-image.
    - `.ppl-card__footer` — Junction + Save styling.

    Hybrid pattern system (Color-specific hatching):
    - `[data-color="teaching"] .ppl-block` — Fine parallel lines (45deg), single repeating-linear-gradient, opacity driven by saturation * 0.4
    - `[data-color="bodyweight"] .ppl-block` — Loose parallel (natural), wider spacing
    - `[data-color="structured"] .ppl-block` — Ruled ledger lines (horizontal)
    - `[data-color="technical"] .ppl-block` — Fine crosshatch (double gradient 45/-45)
    - `[data-color="intense"] .ppl-block` — Bold crosshatch, tight spacing
    - `[data-color="circuit"] .ppl-block` — Diagonal stripes (60deg)
    - `[data-color="fun"] .ppl-block` — Loose diagonal (30deg)
    - `[data-color="mindful"] .ppl-block` — Barely-there horizontal

    Pattern density driven by `var(--ppl-weight-saturation)` using `calc()` in rgba alpha. See RESEARCH.md code examples for exact technique.

    Guilloche header accent: SVG background-image with concentric circles using data URI. Stroke color themed via CSS var reference in the SVG URL-encoded string (or use a fixed dark tone with opacity, since SVG data URIs cannot reference CSS vars — use --ppl-theme-border color extracted at render time and injected into the SVG data URI by the WorkoutCard.ts renderer).

    Typography driven by CSS vars: `font-weight: var(--ppl-weight-font-weight)`, `line-height: var(--ppl-weight-line-height)`, `letter-spacing: var(--ppl-weight-spacing)`.

    Block separators: `.ppl-block + .ppl-block` border-top using `--ppl-theme-border`.
