<task type="auto">
  <name>Task 1: Create canvas-to-production.sh and batch-propagate.sh with Node weight driver</name>
  <files>canvas/scripts/canvas-to-production.sh, canvas/scripts/batch-propagate.sh, canvas/scripts/run-weight.mjs</files>
  <action>
Create three files:

**canvas/scripts/canvas-to-production.sh** — One-way copy with boundary enforcement.
- Header: `#!/usr/bin/env bash` + `set -euo pipefail`
- Usage line: `bash canvas/scripts/canvas-to-production.sh <canvas-artifact-path>`
- Accept one argument: the source path inside `canvas/components/`
- Validate: file exists, path starts with `canvas/components/`
- Route by extension:
  - `.html` → `html/${FILENAME}`
  - `.tsx` or `.ts` → `web/src/components/canvas/${FILENAME}`
  - `.css` → `html/design-system/${FILENAME}`
  - `.md` → exit 1 with message: ".md files require explicit deck path. Use: cp -n $SOURCE cards/<order>/<axis>/<type>/<filename>"
  - anything else → exit 1 "Unknown file type"
- Boundary check: loop over prohibited prefixes (middle-math, scripts, seeds, scl-deep) — if destination starts with any, exit 1
- Create destination directory with `mkdir -p` before copy (html/, web/src/components/canvas/, html/design-system/ may not exist yet)
- Copy with `cp -n` (no-clobber). On failure (file exists), exit 1 with "Destination already exists" message
- On success, echo "COPIED: $SOURCE -> $DEST"

**canvas/scripts/batch-propagate.sh** — Weight engine driver for N zip codes.
- Header: `#!/usr/bin/env bash` + `set -euo pipefail`
- Usage: `bash canvas/scripts/batch-propagate.sh <design-element> <N> [zip-list]`
- Accept 3 args: ELEMENT (required), N (required positive integer), ZIP_LIST (optional comma-separated 4-digit zips)
- If ZIP_LIST not provided as arg 3, check `$ZIP_LIST` env var. If neither, generate first N zips from Deck 07 (2113, 2123, 2133, 2143, 2153 pattern — Order=2, Axis=1, Types 1-5, Color=3)
- First step: build canvas TypeScript: `(cd canvas && npm run build 2>/dev/null)` with a `--skip-build` check: if `$1` is `--skip-build`, shift args and skip the build
- For each zip in the list (up to N): call `node canvas/scripts/run-weight.mjs $ZIP` to get CSS vars JSON, write output to `canvas/components/${ELEMENT}-${ZIP}.html` with a comment header and the CSS vars as a style block
- Count written files, report summary at end
- All writes go to `canvas/components/` only — never to production directories

**canvas/scripts/run-weight.mjs** — Minimal ESM driver.
- Import `resolveZip` from `'../dist/weights/resolver.js'` (relative to script location — use `import.meta.url` to resolve correctly, or use `'./canvas/dist/weights/resolver.js'` if called from repo root)
- Import `weightsToCSSVars` from `'../dist/rendering/index.js'`
- Import tokens from `'../dist/tokens/tokens.js'` (check the actual export name — may need to read the compiled output)
- Read zip from `process.argv[2]`, parse 4 digits: `[o, a, t, c]`
- Call `resolveZip(o, a, t, c)` then `weightsToCSSVars(vec, tokens)`
- Output JSON to stdout: `console.log(JSON.stringify(result, null, 2))`
- Exit 1 with usage message if no zip argument provided

Note on token import: The tokens module was built in Phase 4 via `build-tokens.mjs`. The compiled output path may be `canvas/dist/tokens/tokens.js` or similar — read `canvas/src/tokens/tokens.ts` to confirm the export name before writing the driver. Use the actual export (likely `tokens` or `designTokens`).

IMPORTANT: Run `(cd canvas && npm run build)` before testing the driver to ensure dist/ exists. If build fails, fix TypeScript errors first.

**Step 3: Create .claude/skills/canvas-save/SKILL.md.**
- Create directory `.claude/skills/canvas-save/`
- Write SKILL.md with this exact frontmatter:
```yaml
---
name: canvas-save
description: "Commit current canvas/ state as a named snapshot. Stages canvas/src/, canvas/components/, canvas/tests/, canvas/scripts/ only. Never stages canvas/.local/ or production paths."
disable-model-invocation: true
argument-hint: "[snapshot-name e.g. 'v0-card-shell']"
allowed-tools: Read, Bash, Grep, Glob
---
```

- Skill body workflow (6 steps):
  1. Read `$ARGUMENTS` for the snapshot name. If empty, print usage instructions and stop.
  2. Verify `canvas/.local/` is in `.gitignore`: `grep -q 'canvas/\.local/' .gitignore`. If missing, print ERROR and stop.
  3. Show current branch: `git branch --show-current` — print to user so they see where the snapshot will land.
  4. Stage only canvas/ paths (excluding .local/): `git add canvas/src/ canvas/components/ canvas/tests/ canvas/scripts/`
  5. Check `git status --short` — if nothing is staged, report "Nothing to snapshot" and stop cleanly.
  6. Commit: `git commit -m "canvas-snapshot: $ARGUMENTS"` — report the commit hash and staged file list.

- The skill uses Read/Bash/Grep/Glob only — no Write/Edit needed since this is a git operation.
- The skill does NOT force a specific branch. It reports the branch name so the user can verify.

**Step 4: Verify gitignore works.**
- Run: `git check-ignore canvas/.local/test-file` — should output the path (meaning it is ignored)
- If it does NOT output the path, the gitignore entry is malformed — fix the trailing slash
