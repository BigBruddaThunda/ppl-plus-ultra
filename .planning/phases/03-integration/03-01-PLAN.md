<objective>
Interaction resolver enforcing the constraint hierarchy (Order > Color > Axis > Type) and ParseResult type definition with defaulted_dimensions from day one.

Purpose: computeRawVector() from Phase 2 sums all dial weights but does not enforce hard suppression ceilings from higher-priority dials. The resolver is the additive post-processing layer that pins positions where Order or Color declare weight <= -6, preventing lower-priority affinities from overriding safety constraints (e.g., Gutter never appearing in Restoration workouts). ParseResult is the contract type that all downstream consumers will depend on.

Output: resolveVector() function, ParseResult interface, passing TDD test suite.
</objective>

    - Import ORDERS_WEIGHTS, COLORS_WEIGHTS from weights/index.js and WEIGHT_VECTOR_LENGTH from types/scl.js
    - Clone the raw vector into a new Float32Array
    - For each W position i (1 to 61):
      - Read ORDER's suppressions[i] (from ORDERS_WEIGHTS[orderPos]). If <= -6: pin resolved[i] to that value, continue.

    - Return the resolved vector.
