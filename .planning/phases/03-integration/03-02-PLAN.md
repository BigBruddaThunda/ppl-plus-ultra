  <action>
    RED phase: Write canvas/tests/scorer.test.ts with tests for all behaviors listed above.

    GREEN phase: Create three source files:

    **canvas/src/parser/tokenizer.ts:**
    - normalizeInput(input: string): string — lowercase, strip punctuation (keep hyphens inside words), normalize unicode (NFD + strip combining marks)
    - tokenize(normalized: string): string[] — split on whitespace into unigrams, then generate adjacent bigrams (e.g., "heavy barbell back" -> ["heavy", "barbell", "back", "heavy barbell", "barbell back"])

    **canvas/src/parser/fuzzy.ts:**
    - Build a module-level singleton: flat array ALL_TERMS from dial-keywords.json terms + exercises.json aliases (built once at import time, not per-call)
    - expandToken(token: string, maxDistance?: number): string[] — uses fastest-levenshtein closest() + distance(). If closest match has edit distance <= maxDistance (default 2), return [match]. Otherwise return [token] unchanged.
    - expandPhrase(phrase: string): string[] — uses fuse.js with Fuse instance over exercise aliases. Config: threshold 0.4, minMatchCharLength 3, keys ['name', 'aliases']. Returns matched exercise names/aliases.

    **canvas/src/parser/scorer.ts:**
    - scoreText(input: string, maxCandidates?: number): ParseResult[] — the full pipeline:
      1. normalizeInput() the input
      2. tokenize() into unigrams + bigrams
      3. For each token: expandToken() for typo correction
      4. For each expanded token: look up in dial-keywords.json (exact match on term field)
      5. Also run expandPhrase() on the full input for exercise alias matching — if matched, extract the exercise's scl_types and map to Type dimensions
      6. Accumulate dimension scores: sum affinity_score per DimensionId (1-26)
      7. Bigram matches override unigram matches for the same dimension group (longer match wins)
      8. For collision_prone keywords: only count if no bigram match exists for that dimension group
      9. Select best position per dimension group: Orders (dims 1-7), Axes (dims 8-13), Types (dims 14-18), Colors (dims 19-26) — take argmax within each group
      10. If a group has zero total score: use default position (Order: 3/Hypertrophy, Axis: 1/Basics, Type: 1/Push, Color: 3/Structured) and add to defaulted_dimensions
      11. Call resolveZip() with the 4 selected positions to get the weight vector
      12. Compute confidence: (totalMatchedScore / (inputTokenCount * 8)) clamped to [0, 1]
      13. Build ParseResult with zip (emoji via numericToEmoji), numericZip, positions, weightVector, confidence, defaulted_dimensions, matchedTerms
      14. For ambiguous inputs: generate candidates by trying top-2 scoring positions for ambiguous dimension groups. Return up to maxCandidates (default 5), sorted by confidence descending.

    REFACTOR if needed: ensure module-level singletons for keyword dict and fuzzy matcher are only built once.
  </action>
