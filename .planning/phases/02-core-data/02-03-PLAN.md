<objective>
Mine first-party SCL sources for a fitness keyword dictionary with ~2,500 terms mapped to zip code dimensions.

Purpose: The keyword dictionary is the vocabulary layer that bridges natural language fitness terms to SCL zip code dimensions. When a user says "heavy barbell back work," the keyword dict maps "heavy" to Order:Strength, "barbell" to Color:Structured/Technical/Intense tier range, and "back" to Type:Pull. This dictionary feeds the Phase 3 text scorer.

Output: KeywordEntry TypeScript interface, dial-keywords.json with >= 2,000 entries, and tests.
</objective>

<!-- Target interface (from research): -->
```typescript
export type DimensionId =
  | 1 | 2 | 3 | 4 | 5 | 6 | 7       // Orders
  | 8 | 9 | 10 | 11 | 12 | 13       // Axes
  | 14 | 15 | 16 | 17 | 18          // Types
  | 19 | 20 | 21 | 22 | 23 | 24 | 25 | 26;  // Colors

export interface KeywordEntry {
  term: string;
  dimension: DimensionId;
  affinity_score: number;            // 1-8 positive only
  collision_prone: boolean;
  source: 'first-party' | 'voice-seed';
}
```
