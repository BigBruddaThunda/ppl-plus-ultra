## Core Concepts

**The Zip Code as Spatial Address**
Every idea, block, and district gets an SCL zip code. Standard format (4-dial) for content nodes. District format (6-dial) for nested containers.

**7+1 Order Files as District Containers**
The seven Orders (🐂 Tuscan → 🖼 Palladian) serve as the primary organizational districts. Each Order file is a district container — a zone of related ideas sharing the same loading phase or development character. The +1 is 🧮 Save (the archive district).

**The Ralph Loop as Primary Sorting Process**
Read → Identify → Assign → Sort → Refine → Repeat. Every session in Graph Parti runs a Ralph Loop pass on whatever has accumulated since the last session.

**Figure-Ground as Semantic Tension**
The graph-parti drawing tradition: figure (solid, active, primary) against ground (void, passive, context). In SCL terms: figure = the content, ground = the constraint structure. The Order is ground. The content is figure.

## Relationship to Kernel

Graph Parti uses SCL as its addressing system. The kernel is the shared language. Graph Parti-specific vocabulary (figure, ground, parti, district, node) extends SCL. It does not replace it.

When a Graph Parti concept maps cleanly to an SCL concept, use the SCL term. When it doesn't, add it to `archideck/intake/` for kernel review.
